# Goodday24news Auto Video Maker

브리핑 원문 → 카드뉴스 이미지 → Google Cloud TTS 음성 → MP4 자동 생성 → 유튜브 쇼츠 자동 업로드까지 한 번에 처리하는 웹앱입니다.

## 핵심 기능

- **[변경] AI 카드뉴스 이미지 자동 생성 기능은 현재 꺼져 있습니다** — 관련 버튼은 비활성화 표시로 보이고 클릭해도 동작하지 않습니다. 다시 켜고 싶으면 `index.html`에서 `const AI_IMAGE_GENERATION_ENABLED = false;`를 `true`로 바꾸면 됩니다.
- **[신규] 이미지 일괄 업로드 시 순서 직접 지정** — 여러 장을 한 번에 업로드하면 바로 배정되지 않고, 각 이미지를 몇 장에 넣을지 고르는 패널이 뜹니다. 확인 후 [이 순서로 적용]을 눌러야 반영됩니다.
- **[신규] 전체 MP3 일괄 다운로드** — 각 장표 MP3를 하나씩 내려받던 것에 더해, `전체 MP3 다운로드` 버튼으로 생성된 음성을 한 번에 순차 다운로드할 수 있습니다. 기존 SRT 자막 다운로드와 함께 사용하면 최종 영상이 잘못되더라도 음성/자막 원본을 따로 보관할 수 있습니다.
- 목표 영상 길이(30/40/50/60초)와 TTS 속도(1.0~1.5배)를 지정하면, AI가 장표당 글자수를 역산해서 스크립트를 씁니다. 화면 좌측에 예상 총 길이가 실시간으로 표시됩니다.
- 텔레그램 봇의 최신 브리핑 메시지를 버튼 클릭으로 가져와 원문 입력창에 자동 채우기 (`api/telegram.js`)
- 장표 수를 6~8장 사이에서 선택 가능 — 이슈가 많은 날은 7~8장까지 늘려서 다룰 수 있습니다. 브리핑 원문에 다룰 내용이 부족하면 AI가 요청한 수보다 적게 만들 수도 있습니다.
- **[개선] AI(`gpt-5.4-mini`)가 브리핑 원문을 읽고 장표별 영상 스크립트(장표 제목/화면 설명/내레이션/자막)를 직접 작성** — 기존의 단순 문장 추출 방식 대비 훨씬 자연스럽고 완결된 스크립트가 나옵니다. AI 호출이 실패하면 자동으로 기존 규칙 기반 방식으로 대체됩니다.
- OpenAI `gpt-image-1` 기반 카드뉴스 배경 이미지 자동 생성 기능이 내장되어 있지만 위에 적었듯 현재는 꺼져 있습니다(코드는 `api/image.js`에 그대로 있음). 켜면 업로드된 장표는 자동 생성 대상에서 제외되고, 개별 "AI로 교체" 버튼을 눌러도 한 번 더 확인을 받습니다.
- Google Cloud TTS(한국어 Neural2/Wavenet/Standard)로 장표별 MP3 생성 → 이미지에 맞춰 자동 낭독
- 장표별 MP3 미리듣기/다운로드, SRT 자막 다운로드
- ffmpeg.wasm 기반 MP4 자동 생성 (영상 비율: 1:1 / 9:16 / 16:9)
- 일반 장표 하단 자막 및 `Instagram @goodday24news` 브랜드 삽입
- 마지막 장표에 **`더욱 자세한 뉴스 확인은 https://gooddaynews.store 에서 확인하세요`** 문구를 자동 삽입 (내레이션 + 자막 모두, AI 생성/규칙 기반 생성 모두 적용)
- **[신규] 완성된 MP4를 유튜브 쇼츠로 자동 업로드** (유튜브 API/OAuth 연동)

## 링크 관련 중요 안내

영상(MP4)은 픽셀로 구워진 이미지이기 때문에 영상 안의 텍스트 링크는 클릭할 수 없습니다. 실제로 클릭 가능한 링크가 필요한 곳은 다음과 같이 처리하세요.

- **유튜브**: 이 앱이 업로드 시 설명란(description)에 `https://gooddaynews.store`를 자동으로 넣어줍니다. 유튜브 설명란의 링크는 클릭 가능합니다.
- **인스타그램**: 릴스 설명에는 클릭 가능한 링크를 넣을 수 없으므로, 프로필의 "링크" 항목에 `https://gooddaynews.store`를 등록하고 캡션에는 "프로필 링크 확인"처럼 유도 문구를 넣는 방식을 권장합니다.
- **블로그(Blogspot)**: 카드뉴스를 옮길 때 `<a href="https://gooddaynews.store">` 형태의 실제 링크 태그를 본문에 넣으면 클릭 가능합니다.

## Vercel 배포 및 환경변수

1. 이 폴더를 GitHub 저장소에 업로드합니다.
2. Vercel에서 Import Project를 선택합니다.
3. Environment Variables에 아래 값을 추가합니다.

| 변수명 | 용도 | 필수 여부 |
|---|---|---|
| `OPENAI_API_KEY` | 카드뉴스 이미지 생성(`gpt-image-1`, 현재 비활성화) + AI 스크립트 생성(`gpt-5.4-mini`) | 필수 |
| `GOOGLE_TTS_API_KEY` | Google Cloud Text-to-Speech(한국어 Neural2/Wavenet) 음성 생성 | 필수 |
| `TELEGRAM_BOT_TOKEN` | 텔레그램 봇에서 최신 브리핑 메시지 가져오기 | 텔레그램 연동 사용 시 필수 |
| `TELEGRAM_CHAT_ID` | 특정 채널/채팅으로만 한정해서 가져오고 싶을 때 (여러 채팅에 있는 봇이라면 권장) | 선택 |
| `YT_CLIENT_ID` | 유튜브 업로드용 OAuth 클라이언트 ID | 자동 업로드 사용 시 필수 |
| `YT_CLIENT_SECRET` | 유튜브 업로드용 OAuth 클라이언트 시크릿 | 자동 업로드 사용 시 필수 |
| `YT_REFRESH_TOKEN` | 유튜브 업로드용 리프레시 토큰 (1회 발급) | 자동 업로드 사용 시 필수 |

4. Deploy를 누릅니다.

> `gpt-image-1`은 OpenAI 개발자 콘솔에서 **Organization Verification(조직 인증)**이 완료되어 있어야 사용할 수 있습니다. 이미지 생성이 403으로 실패하면 OpenAI 콘솔에서 인증 상태를 먼저 확인하세요.

## 텔레그램 연동 준비

기존에 모닝브리핑을 발송하던 봇을 그대로 재사용할 수 있습니다.

1. 이미 브리핑을 보내고 있는 봇의 토큰(`@BotFather`에서 발급받은 값)을 `TELEGRAM_BOT_TOKEN`에 등록합니다.
2. 봇이 여러 채널/채팅에 들어가 있어서 특정 채널의 메시지만 가져오고 싶다면, 그 채널/채팅의 숫자 ID를 `TELEGRAM_CHAT_ID`에 등록합니다.
   - 채팅 ID를 모르면, 일단 `TELEGRAM_CHAT_ID`를 비워두고 `텔레그램에서 가져오기`를 한 번 눌러보세요. 상태 메시지에 채팅 이름이 표시되니, 여러 개가 섞여 나오면 그때 채팅 ID를 특정해서 추가하면 됩니다.
3. **주의**: 이 기능은 텔레그램의 `getUpdates` 방식을 사용하는데, 같은 봇 토큰에 webhook이 설정되어 있으면 충돌해서 동작하지 않습니다. 지금 봇이 브리핑을 "보내기"만 하고 별도 webhook을 설정한 적이 없다면 문제없이 동작합니다. 만약 오류가 나면 브라우저 주소창에 `https://api.telegram.org/bot<봇토큰>/deleteWebhook`을 한 번 입력해 webhook을 해제한 뒤 다시 시도하세요.
4. 텔레그램은 봇이 받은(혹은 보낸) 메시지를 최대 24시간까지만 보관합니다. 그 이후에는 `가져오기`로 불러올 수 없으니, 브리핑이 발송된 당일에 사용하세요.

## Google Cloud TTS 연동 준비

1. [Google Cloud Console](https://console.cloud.google.com)에서 프로젝트를 하나 만듭니다(기존 프로젝트가 있으면 그대로 사용해도 됩니다).
2. **API 및 서비스 → 라이브러리**에서 `Cloud Text-to-Speech API`를 검색해 사용 설정합니다.
3. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → API 키**를 클릭해 키를 발급받습니다.
4. (권장) 발급된 키를 클릭해 "API 제한사항"에서 `Cloud Text-to-Speech API`만 허용하도록 제한해두면 키가 유출되어도 다른 API에는 쓰이지 못합니다.
5. 이 키를 Vercel 환경변수 `GOOGLE_TTS_API_KEY`에 등록하고 재배포합니다.
6. 결제 계정 연결이 필요합니다(신규 가입 시 무료 크레딧 제공). Text-to-Speech는 매달 일정 글자 수까지 무료 등급이 있고 이후 종량 과금되며, Standard/Wavenet/Neural2 순으로 단가가 올라갑니다. 정확한 최신 요금은 [Cloud Text-to-Speech 가격 페이지](https://cloud.google.com/text-to-speech/pricing)에서 확인하세요.

## 유튜브 API 연동 준비 (최초 1회만 설정)

유튜브 자동 업로드는 사용자 계정 인증(OAuth)이 필요합니다. 아래 순서로 `YT_CLIENT_ID` / `YT_CLIENT_SECRET` / `YT_REFRESH_TOKEN`을 한 번만 발급받아 Vercel 환경변수에 등록하면, 이후에는 앱에서 버튼 클릭만으로 업로드됩니다.

1. **Google Cloud Console**(console.cloud.google.com)에서 새 프로젝트를 만듭니다.
2. **API 및 서비스 → 라이브러리**에서 `YouTube Data API v3`를 검색해 사용 설정합니다.
3. **API 및 서비스 → OAuth 동의 화면**을 구성합니다. 개인 채널에만 쓸 경우 "테스트 사용자"에 본인 구글 계정을 추가하면 앱 심사 없이 바로 사용할 수 있습니다(단, 이 경우 `공개(public)` 업로드 대신 `비공개` 또는 `미등록`으로 테스트하는 것을 권장합니다).
4. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**를 선택하고, 애플리케이션 유형은 **웹 애플리케이션**으로 만든 뒤, 승인된 리디렉션 URI에 `https://developers.google.com/oauthplayground`를 추가합니다. 여기서 나온 **클라이언트 ID / 클라이언트 보안 비밀**이 각각 `YT_CLIENT_ID`, `YT_CLIENT_SECRET`입니다.
5. **Google OAuth 2.0 Playground**(developers.google.com/oauthplayground)로 이동합니다.
   - 오른쪽 위 톱니바퀴(설정) 클릭 → "Use your own OAuth credentials" 체크 → 위에서 만든 클라이언트 ID/시크릿 입력.
   - 왼쪽 목록에서 `YouTube Data API v3` → `https://www.googleapis.com/auth/youtube.upload` 스코프 선택 → **Authorize APIs** 클릭 → 본인 유튜브 계정으로 로그인/동의.
   - **Exchange authorization code for tokens** 클릭 → 여기서 나오는 **Refresh token** 값이 `YT_REFRESH_TOKEN`입니다.
6. 발급받은 3개 값을 Vercel 프로젝트의 Environment Variables에 등록하고 재배포합니다.

### 유튜브 업로드 시 참고 사항

- YouTube Data API는 **일일 쿼터 10,000 유닛**이 기본 제공되며, 업로드 1회당 1,600 유닛이 소모됩니다(하루 약 6회 업로드 가능).
- 쇼츠로 인식되려면 **세로 9:16, 60초 이하**가 안전하며, 제목이나 설명에 `#Shorts`를 포함해야 합니다(이 앱은 기본 제목에 자동으로 넣어줍니다).
- OAuth 앱이 "테스트" 상태인 동안에는 `공개(public)` 업로드가 제한될 수 있으니, 먼저 `미등록(unlisted)` 또는 `비공개(private)`로 테스트해보세요.
- 브라우저가 구글 업로드 서버로 영상을 직접 전송하는 방식이라 Vercel 서버 용량 제한과 무관하게 동작합니다. 서버(`/api/youtube-token`)는 단명 액세스 토큰 발급만 담당하고, 클라이언트 시크릿과 리프레시 토큰은 브라우저로 절대 전달되지 않습니다.

## 사용 순서

1. 좌측 설정에서 음성/TTS 속도(예: 1.15배)를 정하고, 우측 브리핑 영역에서 장표 수와 목표 영상 길이(예: 40초)를 정합니다.
2. `텔레그램에서 가져오기`를 눌러 오늘 발송된 브리핑 메시지를 자동으로 불러오거나, 브리핑 원문을 직접 붙여넣습니다.
3. `AI로 스크립트 생성`을 누릅니다. (마지막 장표에는 `gooddaynews.store` 안내 문구가 자동으로 붙습니다.)
4. 화면 왼쪽의 "예상 총 길이" 표시를 확인합니다. 60초를 넘으면 TTS 속도를 올리거나, 목표 길이를 줄여서 다시 생성하세요.
5. `이미지 일괄 업로드`로 카드뉴스 이미지를 한 번에 선택하면, 어떤 이미지를 몇 장에 넣을지 지정하는 패널이 뜹니다. 확인 후 `이 순서로 적용`을 누르세요. (한 장씩 넣으려면 장표별 썸네일을 개별 클릭해도 됩니다.) AI 이미지 자동 생성은 현재 꺼져 있습니다.
6. `전체 MP3 생성`을 눌러 이미지에 맞춰 낭독 음성을 만듭니다. (스크립트를 다시 만들거나 속도를 바꿨다면 꼭 다시 생성하세요.)
7. 필요하면 `전체 MP3 다운로드`와 `SRT 자막 다운로드`로 음성/자막 원본을 미리 백업해두세요.
8. `MP4 생성`을 눌러 영상을 완성합니다. 완료되면 다운로드가 자동으로 시작됩니다.
9. (선택) 하단 `유튜브 쇼츠 자동 업로드` 영역에서 제목/설명/태그를 확인하고 `유튜브 쇼츠 업로드`를 누릅니다.

## 참고

- MP4 생성은 브라우저 내부의 `ffmpeg.wasm`으로 처리됩니다.
- `ffmpeg.wasm` 코어/워커 파일은 CDN(unpkg)에서 받아와 `toBlobURL`로 "같은 출처"인 것처럼 변환해서 사용합니다(코어뿐 아니라 `@ffmpeg/ffmpeg`의 자체 워커 스크립트도 동일하게 처리). 이렇게 하지 않으면 브라우저가 "Failed to construct 'Worker': ... cannot be accessed from origin" 오류를 내며 차단합니다. **자체 호스팅(레포에 파일 직접 포함)은 시도해봤지만 GitHub 웹사이트 업로드가 파일당 25MB 제한이 있어서(핵심 wasm 파일이 31MB) 오히려 배포가 깨지는 원인이 되므로 다시 CDN 방식으로 되돌렸습니다.** git 명령어나 GitHub Desktop으로 push하는 환경이라면 자체 호스팅도 가능하니 필요하면 요청하세요.
- 최초 실행 시 ffmpeg.wasm CDN 다운로드 때문에 시간이 걸릴 수 있습니다.
- 브라우저 메모리 제한이 있으면 1:1 비율부터 테스트하세요.
- OpenAI API 키, Google TTS API 키, 유튜브 클라이언트 시크릿/리프레시 토큰은 브라우저에 노출되지 않고 Vercel 서버리스 함수에서만 사용됩니다.

## v10 수정 사항

- **v9의 ffmpeg.wasm 자체 호스팅을 되돌렸습니다.** GitHub 웹사이트에서 드래그앤드롭으로 업로드하는 배포 방식에서는 파일당 25MB 제한이 있어서, 31MB짜리 `ffmpeg-core.wasm`이 정상적으로 올라가지 않아 오히려 "ffmpeg.wasm 모듈이 아직 로딩되지 않았습니다" 오류가 발생했습니다. 다시 CDN(unpkg) + `toBlobURL` 방식으로 되돌리고, `classWorkerURL`까지 blob 처리하는 수정(v6)은 그대로 유지했습니다. git 명령어/GitHub Desktop으로 push하는 환경이라면 자체 호스팅이 더 안정적이니 필요하면 다시 요청하세요.

## v9 수정 사항

- **ffmpeg.wasm 자체 호스팅으로 전환**: 기존에는 unpkg CDN에서 코어/워커 파일을 받아 `toBlobURL`로 우회하는 방식이었는데, 이 구조 자체가 "Failed to construct 'Worker'" 류의 크로스오리진 오류가 재발할 여지를 남겨뒀습니다. 이번에 `@ffmpeg/ffmpeg`, `@ffmpeg/core` 패키지 파일을 프로젝트 안(`/vendor`)에 직접 포함해서 같은 출처에서 서빙하도록 바꿨습니다. 이제 CDN도, blob 변환도, COOP/COEP 헤더도 필요 없습니다 — 구조적으로 해당 오류가 다시 날 수 없습니다. (참고로 경로가 절대경로(origin 기준)로 두 번 겹쳐 해석되는 미묘한 버그도 이 과정에서 함께 잡았습니다.)
- `vercel.json`에서 더 이상 필요 없는 COOP/COEP 헤더를 제거하고, `/vendor` 정적 파일에 1년 캐시 헤더를 추가했습니다.
- `@ffmpeg/util`(fetchFile) 의존성을 제거하고 동일한 기능을 3줄짜리 로컬 함수로 대체해 외부 의존성을 하나 더 줄였습니다.

## v8 수정 사항

- **MP4 저장 안정성 강화**: 생성된 파일이 0바이트인 경우를 명확한 오류로 표시하고, 생성이 끝나면 다운로드가 자동으로 시작되도록 변경(기존에는 링크를 직접 클릭해야 했습니다). 실패 시 원인 추정 안내 문구도 더 구체적으로 바꿨습니다.
- **TTS 속도 조절 기능 추가**(`ttsSpeed` 셀렉트, 1.0~1.5배) — `api/tts.js`가 OpenAI TTS의 `speed` 파라미터를 지원합니다.
- **근본 원인 수정 — 스크립트 길이가 목표 영상 길이를 반영하지 않던 문제**: 기존에는 장표당 130~220자로 고정돼 있어, 6장만 있어도 1x 속도 기준 2.5~4분짜리 영상이 나왔습니다(쇼츠 규격 초과 + MP4 생성 실패의 원인). 이제 "목표 영상 길이"(30/40/50/60초)를 지정하면 장표 수·TTS 속도까지 반영해 장표당 글자수를 역산하여 AI에게 지시합니다. 목표 길이가 짧을수록(30~40초) 내레이션이 한 문장 수준으로 짧아진다는 점을 프롬프트에도 명시했습니다.
- 화면에 "예상 총 길이"를 실시간으로 표시(60초 초과 시 빨간색으로 경고). 규칙 기반 폴백(`autoBuildFromBriefingLocal`)도 목표 길이에 맞춰 스크립트를 자동으로 잘라냅니다.
